# MiniMax Speech API: Руководство по оптимизации русского языка

Чтобы добиться максимального качества русской речи с правильной грамматикой (просодией) и ударениями при использовании MiniMax API, следуйте этим рекомендациям по настройке.

## 1. Основные настройки API

Для русского языка необходимо явно указать модели, что приоритетными являются русские фонетические правила и интонация.

| Параметр | Рекомендуемое значение | Обоснование |
| :--- | :--- | :--- |
| **`model`** | `speech-2.8-hd` | Последние HD-модели обладают лучшей контекстно-зависимой просодией и поддерживают более естественные выражения. |
| **`language_boost`** | `"Russian"` | **Критически важно.** Гарантирует, что модель будет обрабатывать ввод как русский, а не пытаться «англизировать» произношение. |
| **`voice_id`** | (Выберите нативный голос) | Убедитесь, что вы используете голос, откалиброванный для многоязычной или конкретно славянской фонетики. |

## 2. Работа с ударениями

Модели MiniMax 2.8 хорошо учитывают контекст и обычно правильно расставляют ударения на основе окружающего текста. Для ручного исправления «упрямых» слов используйте `pronunciation_dict`.

### Использование `pronunciation_dict`
Если слово, например *пр+авильно*, синтезируется неверно, добавьте его в словарь:

```json
{
  "voice_setting": { ... },
  "pronunciation_dict": {
    "tone": ["ко́мпас/ко+мпас", "атлас/атл+ас"]
  }
}
```

> [!TIP]
> Хотя MiniMax 2.8 достаточно умён, если вы обнаружите ошибки в конкретных словах, попробуйте заменить их в словаре на версию с символом `+` перед ударной гласной (например, `замок/зам+ок` для крепости против `замок/з+амок` для запорного устройства).

## 3. Улучшение фразировки и просодии

В TTS под фразировкой понимается то, как голос делает паузы и расставляет акценты в предложении.

- **Пунктуация имеет значение**: Убедитесь, что в тексте четко расставлены запятые, точки и вопросительные знаки. Модель использует их как основные триггеры для изменения интонации.
- **Естественность**: Модель MiniMax 2.8-hd самостоятельно анализирует контекст предложения для создания естественных вдохов и пауз.

> [!IMPORTANT]
> Мы намеренно не используем автоматическую вставку тегов (вздохи, смех и т.д.), чтобы избежать искажения смысла и сохранить профессиональный тон озвучки.

## Пример тела запроса (Request Body)

```json
{
  "model": "speech-2.8-hd",
  "text": "Здравствуйте! Сегодня мы изучим, как правильно настраивать MiniMax для русского языка. Это очень просто!",
  "voice_setting": {
    "voice_id": "Russian_Engaging_Podcaster_v1",
    "speed": 1.1,
    "vol": 1.0,
    "pitch": 0
  }
}
```

---
> [!TIP]
> Для получения списка доступных голосов или использования Custom ID, воспользуйтесь соответствующим полем в настройках аватара в интерфейсе приложения.
